### 2. 개발 전략 및 원칙 확정
- **문서화:** "끝나고 한꺼번에" 하지 않고, **지금부터 매일 조금씩** 작성
  - `History/`: 날짜별 구현 로그 (현재 진행 중)
  - `Docs/`: 아키텍처, 설계 결정, RPC 흐름도 등 핵심 기술 문서 (별도 폴다 생성 권장)
  - 이유: 5주 후면 구현 디테일 50% 이상 망각. 지금 메모하는 게 훨씬 빠름.
- **영상 촬영:** 기능 단위 30초~1분 클립 다수 → 나중에 3~5분 하이라이트 + 숏폼 편집
  - 인벤토리 장착 → 스탯 변화 실시간 확인
  - 스킬 사용 → 쿨타임 → HUD 연동
  - 로비 → 던전 입장 → 서버 이동
- **Scope Creep 방지:** "완성 먼저, 완벽은 나중에"
  - 스킬: 3~5개 핵심만 (기본 공격 + 2액티브 + 1궁극기)
  - 웹서버: 로그인 + 인벤토리/스탯 저장·불러오기 MVP만
  - 로비: 혼자 입장 버튼으로 단순화 (매치메이킹 Phase 2)

### 3. 기술적 고도화 후보 기능 정리

| 기능 | 난이도 | 어필 포인트 |
|------|--------|-------------|
| **Gameplay Prediction** | ⭐⭐⭐⭐ | 서버 지연 100ms에서도 부드러운 스킬 시전 |
| **Ability Queue** | ⭐⭐⭐ | 쿨타임 중 입력 → 종료 즉시 발동 |
| **AI Behavior Tree + GAS** | ⭐⭐⭐⭐ | 몬스터도 GAS 스킬/버프/디버프 사용 |
| **Hitbox/Hurtbox 다형 콜리전** | ⭐⭐⭐ | 근접 공격 정확한 판정 |
| **Save/Load + WebServer** | ⭐⭐⭐⭐ | 클라우드 세이브, 계정별 영속화 |
| **Dedicated Server 클러스터링** | ⭐⭐⭐⭐⭐ | 여러 던전 인스턴스 동시 운영 |

- **특히 Gameplay Prediction:** 멀티플레이어 프로젝트에서 "이 사람 진짜 안다"는 걸 보여주는 가장 좋은 기능. Tom Looman 강의에도 다뤄질 예정.

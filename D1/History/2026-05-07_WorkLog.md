# 작업 로그: 2026-05-07

## 작업 내용

### 1. 아이템 사용 기능 완성 (C++ GE 연동)
- **파일:** `Source/D1/Inventory/D1InventoryComponent.cpp`
- **구현:**
  - `UseItemInternal()`에 실제 아이템 사용 로직 구현
  - 소유자(PlayerState)의 ASC(`UAbilitySystemComponent`) 획득
  - `GameState`에서 `ItemRegistry`를 통해 `UD1ItemData` 룩업
  - `ItemType == Consumable`인지 확인 후 `UseEffect` GE를 ASC에 `ApplyGameplayEffectSpecToSelf`
  - GE 적용 후 `RemoveItem(SlotIndex, 1)`로 아이템 1개 소모
  - 장비 아이템 등 소비 불가 아이템은 조기 리턴 및 로그 출력

### 2. WBP_ItemDetail UI 연동
- **파일:** Blueprint 에셋 (`WBP_ItemDetail`, `WBP_Inventory`, `WBP_Button_Use`)
- **구현:**
  - `WBP_ItemDetail`에 `CurrentSlotIndex` 변수 추가 및 `UpdateDetail` 함수 파라미터 확장
  - `WBP_Button`의 `OnMyButtonClicked` Event Dispatcher 활용 (IoC 패턴)
  - `WBP_ItemDetail` → `OnUseRequested` Event Dispatcher 추가
  - `WBP_Inventory`에서 `OnUseRequested`를 Bind하여 `InventoryWidgetController.UseItem(SlotIndex)` 호출
  - 사용 버튼 클릭 시 팝업이 자동으로 닫히도록 처리

### 3. 초기화 시점 이슈 해결
- **문제:** `Event Construct`에서 `Get Inventory Widget Controller` 호출 시 `Accessed None` 에러 발생
- **원인:** `Event Construct`는 위젯 생성 직후 실행되므로 `WidgetController`가 아직 설정되지 않았을 수 있음
- **해결:** 초기화 로직을 `Event Construct`가 아닌 **`Set Widget Controller` 이후 시점**으로 이동
  - `D1UserWidget`의 `SetWidgetController` 이후 호출되는 이벤트/함수에서 컨트롤러 접근하도록 수정

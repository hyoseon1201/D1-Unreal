## 2. 산출물

### 2-1. `GA_Focus` (버프 Ability) — 완성
- **GE:** `GE_Focus` (Duration GE, 10초)
  - Modifier: `AttackPower` = Add 30
  - Granted Tags: `Status.Buff.Focus`
  - Duration Policy: Has Duration, Period: Infinite
- **Cooldown:** `GE_Cooldown_Focus` (12초, Instant, `Cooldown.Buff.Focus`)
- **BP 구조:** `CommitAbility` → `HasAuthority` → `ApplyGameplayEffectToOwner` + `PlayMontageAndWait` 동시 실행
- **GameplayCue:** `GameplayCue.Buff.Focus` 태그 등록 (향후 지속 오라용)
- **Input Block:** `ActivationOwnedTags`에 `Player.Block.InputPressed`, `Player.Block.InputHeld` 적용

### 2-2. `GA_Heal` (자기 회복 Ability) — 완성
- **GE:** `GE_Heal` (Instant)
  - Modifier: `Health` = Add 200
- **Cost:** `GE_Cost_Mana_Heal` (Mana -30, Instant)
- **Cooldown:** `GE_Cooldown_Heal` (10초)
- **BP 구조:** `CommitAbility` → `HasAuthority` → `ApplyGameplayEffectToOwner` + `PlayMontageAndWait`
- **애니메이션:** 기존 애니메이션 재활용한 짧은 캐스팅 몽타주

### 2-3. Secondary Init GE 구조 개선
- **문제:** 기존 Secondary Init GE가 Infinite Duration + Override라서, Secondary Attribute에 Add Modifier(장비, 버프)가 덮여서 적용되지 않았음.
- **해결:** `GE_SecondaryAttributes`를 **Infinite → Instant Duration**으로 변경.
- **결과:** 장비 버프와 스킬 버프(Focus의 AttackPower +30)가 Secondary Attribute에 정상적으로 Add됨.
- **주의사항:** Init GE가 Instant이므로, Primary Attribute 변경(레벨업/스탯 투자) 시 Secondary Base Value를 재계산해야 함.

### 2-4. `RecalculateSecondaryAttributes` 함수 추가
- **위치:** `D1AbilitySystemLibrary.h/.cpp`
- **역할:** Secondary Init GE만 다시 적용하여 Base Value를 재계산. 기존 Add Modifier(장비, 버프)는 그대로 유지됨.
- **호출 지점:** `D1AbilitySystemComponent::ServerUpgradeAttribute_Implementation` (스탯 투자 후)
- **동작:**
  ```
  Primary Attribute 변경 (예: STR +1)
  → Secondary Init GE 재적용 (Base Value만 새로 계산)
  → 기존 Add Modifier(장비 +50, 버프 +30)는 그대로 유지
  → 결과: Armor Base Value 재계산 + 50 + 30
  ```

## 4. 주요 기술 결정

### 4-1. Secondary Attribute 버프 방식 확정
- **Instant Init GE 방식 채택:** Secondary Attribute에 직접 Add Modifier를 걸 수 있음.
- **재계산 전략:** Primary 변경 시 `RecalculateSecondaryAttributes()` 호출로 Base Value만 갱신.
- **이전 방식(Primary 우회) 폐기:** 장비 버프도 Secondary에 직접 걸도록 원상복구 예정.

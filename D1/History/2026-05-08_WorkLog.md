# 작업 로그: 2026-05-08

## 작업 내용

### 1. 장비 장착/탈착 시스템 C++ 구현
- **파일:** `Source/D1/Inventory/D1InventoryComponent.h/.cpp`, `Source/D1/Inventory/D1InventoryTypes.h`, `Source/D1/UI/WidgetController/D1InventoryWidgetController.h/.cpp`
- **구현:**
  - `FD1EquippedItem` 구조체 신규 생성 (`EEquipmentSlot` + `FD1InventoryItem`, Replication 지원용)
  - `EquippedItems` (`TArray<FD1EquippedItem>`): 현재 장착 중인 장비 목록 (Replicated)
  - `EquippedEffectHandles` (`TMap<EEquipmentSlot, FActiveGameplayEffectHandle>`): 장착된 GE의 Active Handle (서버 전용, 탈착 시 제거용)
  - `ServerEquipItem(int32 InventorySlotIndex)` / `ServerUnequipItem(EEquipmentSlot Slot)` RPC 추가
  - `EquipItemInternal()`:
    - ItemRegistry에서 메타데이터 룩업
    - `ItemType == Equipment` 확인 및 `EquipmentSlot` 유효성 검사
    - 해당 부위에 이미 장비가 있으면 `UnequipItemInternal()`로 자동 탈착
    - `EquipEffect` GE를 ASC에 `ApplyGameplayEffectSpecToSelf` → Handle 저장
    - 인벤토리에서 1개 제거 → `EquippedItems`에 추가
  - `UnequipItemInternal()`:
    - ASC에서 `RemoveActiveGameplayEffect(Handle)`로 GE 제거
    - 인벤토리에 `AddItem()`으로 되돌리기 (빈 슬롯 없으면 Drop 처리 예정)
    - `EquippedItems`에서 제거
  - `OnEquippedItemsChanged` 델리게이트 추가 (클리언트 동기화 및 UI 갱신용)

### 2. WidgetController에 장비 관련 함수 추가
- **파일:** `Source/D1/UI/WidgetController/D1InventoryWidgetController.h/.cpp`
- **구현:**
  - `EquipItem(int32 SlotIndex)`: `ServerEquipItem` RPC 호출
  - `UnequipItem(EEquipmentSlot Slot)`: `ServerUnequipItem` RPC 호출
  - `OnEquippedItemsUpdatedSignature` 델리게이트 추가 (`TArray<FD1EquippedItem>` 파라미터)
  - `BindCallbacksToDependencies()`에서 `OnEquippedItemsChanged`에 동적 바인딩
  - `BroadcastInitialValues()`에서 장착 장비 초기 상태도 Broadcast

### 3. GameplayEffect 장착 테스트 준비
- **에디터 작업:** `GE_Equip_Sword_Iron` Blueprint 생성 필요
  - Duration Policy: `Infinite`
  - Modifiers: `AttackPower + 15`, `MaxHealth + 20` 등 Secondary Attributes 직접 수정
- **플레이어 테스트 아이템:** `PlayerState::BeginPlay`에 `Sword_Iron` 추가하여 장착 테스트 준비

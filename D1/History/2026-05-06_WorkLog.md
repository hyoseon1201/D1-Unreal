# 작업 로그: 2026-05-06

## 작업 내용

### 1. GameState 기반 아이템 룩업 구조 개선
- **파일:** `Source/D1/Game/D1GameStateBase.h/.cpp`, `Source/D1/Game/D1GameModeBase.h/.cpp`, `Source/D1/AbilitySystem/D1AbilitySystemLibrary.cpp`
- **문제:** 클라이언트에서 `GetItemData()` 호출 시 `GetGameMode()`가 NULL을 반환하여 아이템 아이콘을 불러올 수 없었음
- **해결:**
  - `AD1GameStateBase` 신규 생성 (`ItemRegistry` 필드 보유)
  - `GameModeBase` 생성자에서 `GameStateClass = D1GameStateBase` 설정
  - `PostInitializeComponents`에서 GameMode의 `ItemRegistry`를 GameState로 복사
  - `GetItemData` 난부 로직을 `GetGameMode` → **`GetGameState`** 기반으로 변경
  - 클라이언트/서버 모두에서 아이템 메타데이터 접근 가능해짐

### 2. 인벤토리 UI 상호작용 구현
- **파일:** Blueprint 에셋들 (`WBP_Inventory`, `WBP_InventorySlot`, `WBP_ItemDetail`)
- **구현 내용:**
  - `WBP_InventorySlot`: `SizeBox` 루트에 `OnMouseButtonDown` 이벤트 연결
  - 좌클릭 → `OnSlotClicked` Event Dispatcher 발동 → `WBP_ItemDetail` 표시
  - 우클릭 → `UseItem(SlotIndex)` 즉시 사용 (예정)
  - `Image_Icon`, `Text_Count`를 `Hit Test Invisible`로 설정하여 루트 이벤트가 가로채지 않도록 처리
  - `WBP_ItemDetail`: 인벤토리 크기와 동일한 **모달(Modal) 팝업** 구조 구현
    - `Image_Background`: 반투명 검정 배경 (Alpha 0.7~0.9)
    - `Border_Panel` + `VerticalBox`로 아이템 정보 레이아웃 구성
    - 아이템 아이콘, 이름, 설명, 사용/장착/버리기 버튼 배치
    - 닫기 버튼 및 배경 클릭으로 `Collapsed` 처리
  - `Event Dispatcher` (`OnSlotClicked`) 기반으로 슬롯-인벤토리 간 느슨한 결합 구현

### 3. 인벤토리 동적 생성 구조 확정
- **설정:** 60개 슬롯 (한 줄 6개, 총 10줄)
- `Event Construct`: `SetUniformGridPanel()` 함수로 60개 슬롯 동적 생성
- `RefreshInventory`: `OnInventoryUpdated` 델리게이트 수신 시 슬롯 아이콘/개수 갱신
- `BroadcastInitialValues`: 위젯 초기화 시 현재 인벤토리 상태 강제 싱크

### 4. 테스트 결과
- **서버:** `BeginPlay`에서 `AddItem("Potion_Health_Small", 5)` → 클라이언트로 정상 Replication
- **클이언트:** 인벤토리 열기 → 0번 슬롯에 포션 아이콘 5개 정상 표시
- **상세 정보:** 좌클릭 시 `WBP_ItemDetail`이 반투명 모달로 표시되며 아이템 이름/아이콘/설명 출력 확인
- **닫기:** 닫기 버튼 클릭 시 디테일 패널이 정상적으로 사라짐

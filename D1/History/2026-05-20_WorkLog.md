## 2. 기술 결정 및 구현 상세

### 2-1. 이동 방식: CMC Velocity 기반 (최종 채택)

**시도했던 방식들:**
1. `LaunchCharacter` → 관성/마찰로 거리 불규칙, Z축 이탈
2. `SetActorLocation` + Timer(Lerp) → CMC 보간 무시로 "순간이동"처럼 끊김
3. `SetActorLocation` + `NetUpdateFrequency=60` → 여전히 CMC 예측 이동 깨짐
4. **최종: `CMC->Velocity = Direction * Speed`** → 서버가 velocity만 설정, 클라이언트는 CMC의 예측 보간으로 자연스럽게 따라감 (정석)

**최종 코드 구조 (`UDashToLocation`):**
- `Activate()`:
  - 서버: `MoveComp->Velocity = DashDirection * DashSpeed`, `SetMovementMode(MOVE_Flying)`, 마찰/감속 0
  - 클라이언트: `bIsDashing = true`만 설정 (delegate 수신용)
  - 둘 다: `SetTimer(Duration, false)` → Duration 후 `FinishDash()`
- `FinishDash()`:
  - 서버: 이동 거리 체크(70% 미만이면 HitWall), `Velocity=0`, 물리 상태 복구
  - 둘 다: `OnDashComplete` 또는 `OnHitWall` delegate 발송

**Flying vs Walking:**
- `MOVE_Flying`으로 지형/경사/콜리전 영향 차단. Walking + 마찰 0만으로는 계단/경사에서 거리가 달라질 수 있음.
- 대시 종료 후 `OriginalMovementMode`로 복구 (OnDestroy에서도 Authority 체크 후 복구).

### 2-2. 네트워크 동기화

| 구분 | 서버 | 클라이언트 |
|------|------|-----------|
| 이동 로직 | `CMC->Velocity` 설정, 물리 상태 변경 | 없음 (서버 위치를 Replicate로 받음) |
| Timer | `Duration` 후 `FinishDash` 호출 | 동일하게 `Duration` 후 `FinishDash` 호출 |
| Delegate | `OnDashComplete` 발송 | `OnDashComplete` 발송 (Montage Jump 등 로컬 처리용) |

→ 클라이언트도 `OnDashComplete`를 받아야 `Montage Jump to Section` 등 로컬 애니메이션 제어가 가능함. 이를 위해 `Activate()`에서 클라이언트를 early return하지 않고 `bIsDashing = true` + Timer를 함께 실행.

### 2-3. 몽타주 구조

`AM_ChargeDash` Section 구조:
- **Charge:** 돌진 자세 (몽타주 시작과 동시에 재생)
- **Slam:** 내리치기 (`Next Section = None`, `AN_SlamImpact`에서 데미지)
- **Recover:** 원래 자세 복귀 (현재 사용 안 함, `Slam`의 `Next=None`으로 바로 종료)

**동작 흐름:**
1. `ActivateAbility` → `SetActorRotation` (Teleport=true, 마우스 방향 즉시 회전)
2. 동시에 `CreateDashToLocation` 실행 (대시 시작)
3. 동시에 `PlayMontageAndWait` 실행 (Charge 섹션 재생)
4. `Duration` 후 `OnDashComplete` → `Montage Jump to Section` (Slam)
5. `Slam` 섹션 재생 중 `AN_SlamImpact` → `CauseDamage`
6. `Slam` 끝 → `PlayMontageAndWait.OnCompleted` → `EndAbility`

→ `OnDashComplete`에는 **절대 `EndAbility`를 연결하지 않음**. 대시가 끝났다고 Ability가 끝나면 Slam 애니메이션이 재생되지 않음.

### 2-4. 입력 블록

**문제:** `ActivationOwnedTags`에 `Player.Block.InputPressed`를 넣어도, 우클릭 AutoRun은 `D1PlayerController`에서 직접 처리되므로 태그를 무시하고 이동함.

**해결:** `D1PlayerController.cpp` 두 곳 수정:
1. `AbilityInputTagReleased()` (RMB일 때): `Player_Block_InputPressed` 태그가 있으면 AutoRun 시작 안 함 (`return`)
2. `AutoRun()`: 실행 시작 시 `Player_Block_InputPressed` 또는 `Player_Block_InputHeld` 태그가 있으면 `bAutoRunning = false` → 즉시 중단

→ Ability 실행 중에는 우클릭(짧게/길게) 및 AutoRun이 완전히 차단됨.

### 2-5. 벽 충돌 체크

- `Sweep`이나 `LineTrace`로 매 틱 체크하면 CMC와 타이밍이 어긋나 오진 가능성.
- **최종 방식:** Duration 후 실제 이동 거리가 설정 거리의 70% 미만이면 "벽에 막힘"으로 판단 (`OnHitWall`).
- 이 방식은 Timer를 1회만 쓰므로 프레임과 무관하게 안정적.

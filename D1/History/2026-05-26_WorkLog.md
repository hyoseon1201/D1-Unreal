# 2026-05-26 작업 로그

## 작업 목표
- `ClientTravel` (Town ↔ Dungeon) 시 `PlayerState` 데이터가 리셋되는 버그 해결
- Primary Attribute (Strength/Int 등) 및 `AttributePoints`가 맵 이동 후에도 유지되도록 구현

## 문제 상황
1. **PIE `ClientTravel`로 Dungeon 이동 시:**
   - 새 `PlayerState` 객체 생성 (`CONSTRUCTOR` 로그 확인)
   - `AttributePoints`가 5→0 또는 5→5로 돌아감 (투자한 포인트 반영 안 됨)
   - `bAbilitySystemInitialized`가 `FALSE`로 초기화 → `InitializeDefaultAttributes()`가 다시 실행됨
   - Primary Attribute(Strength 등)가 기본값으로 리셋됨
2. **원인:** PIE/Standalone에서는 `ClientTravel`이 기존 PlayerState를 유지하지 않고 **새로 생성**함
3. **초기 접근법:** `bAbilitySystemInitialized`를 `Replicated`로 설정했으나, PIE에서는 의미 없음 (새 객체 생성)

## 해결 방안 (GameInstance 임시 저장 패턴)
### 1. `UD1GameInstance` 신규 생성 (`Game/D1GameInstance.h/.cpp`)
- `SavePlayerStateData(bInit, AttrPts, Level, XP, Str, Int, Dex, Luc)`: Travel 직전 데이터 저장
- `RestorePlayerStateData(...)`: Travel 후 데이터 복원
- `ClearSavedData()`: 복원 후 클리어
- `HasSavedData()`: 저장 여부 확인

### 2. `AD1PlayerController` 수정 (`Player/D1PlayerController.h/.cpp`)
- `PreClientTravel(const FString& PendingURL, ETravelType TravelType, bool bIsSeamlessTravel)` 오버라이드 추가
  - Engine이 `ClientTravel` 직전에 **자동으로** 호출하는 함수 (BP_DungeonPortal이 `ClientTravel` 직접 호출해도 작동)
  - `PlayerState` + `AttributeSet` 데이터를 `GameInstance`에 저장
- `TravelToMap`에서는 별도 저장 로직 제거 (PreClientTravel이 처리)

### 3. `AD1PlayerState` 수정 (`Player/D1PlayerState.h/.cpp`)
- `RestoreTravelDataIfNeeded()` 함수 추가
  - `PossessedBy`에서 호출되도록 설계 (초기화보다 먼저)
  - `bAbilitySystemInitialized` 복원 → `InitializeDefaultAttributes()` 스킵 여부 결정
  - `AttributePoints`, `Level`, `XP` 복원
  - `Primary Attribute` (Strength/Int/Dex/Luc) 직접 `Set` 복원
  - `OnRep_*` 호출로 UI/Delegate 갱신

### 4. `AD1Hero` 수정 (`Characters/D1Hero.cpp`)
- `PossessedBy`에서 `InitAbilityActorInfo()` 직후 `D1PS->RestoreTravelDataIfNeeded()` 호출
- **복원 성공 시 (`bInit=TRUE`):**
  - `InitializeDefaultAttributes()` **스킵** (Primary GE 덮어쓰기 방지)
  - 대신 `Secondary GE + Vital GE`만 **재적용** (`ApplyEffectToSelf`)
  - 새 `AttributeSet`은 Secondary/Vital GE가 필요함
- **복원 실패 시 (최초 접속):**
  - `InitializeDefaultAttributes()` + `AddCharacterAbilities()` 정상 실행

### 5. `AD1GameModeBase` 수정 (`Game/D1GameModeBase.h/.cpp`)
- `PostLogin` 오버라이드 추가 → Travel 후 PlayerState 상태 디버그 로그

### 6. `DefaultEngine.ini` 수정
- `GameInstanceClass=/Script/D1.D1GameInstance` 추가
- **주의:** Project Settings → Maps & Modes → Game Instance Class에서도 `D1GameInstance` 선택 필요

## 테스트 결과
- **Town → Dungeon 이동:**
  - `PreClientTravel`: `bInit=TRUE, AttrPts=0, Str=11.0, Int=5.0...` 저장 확인
  - `PossessedBy`: `RestoreTravelDataIfNeeded` → `bInit=TRUE` 복원 → `InitializeDefaultAttributes` 스킵
  - `BeginPlay`: 이미 복원 완료된 상태
  - **결과:** AttributePoints=0, Strength=11.0 정상 유지 확인 ✅

## 참고 사항
- `PreClientTravel`은 Engine의 `APlayerController::ClientTravel` 직전에 항상 호출됨

- `Cast<UD1AttributeSet>` 필요: `GetAttributeSet()`이 `UAttributeSet*`를 반환함
- Secondary Init GE는 **Instant Duration** (장비/버프 Add Modifier 충돌 방지)
